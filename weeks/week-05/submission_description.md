In Week 5, I implemented a robust model evaluation and risk assessment layer to ensure the predictive work is suitable for food-quality decision-making. In quality-control contexts, evaluation must extend beyond a single metric: false negatives can be safety-critical, and models must remain reliable across product segments and over time. This week therefore focuses on performance measurement, calibration, stability, and drift awareness.

The Week 5 script evaluates each trained baseline model on a held-out test set and exports a consistent metrics table (including ROC-AUC and PR-AUC, plus precision, recall, and F1 at a defined operating threshold). The executed results show clear trade-offs between the two baseline approaches: the random forest achieved **ROC-AUC 0.873** and **PR-AUC 0.626** (higher ranking performance), while logistic regression achieved **ROC-AUC 0.819** and **PR-AUC 0.473**. At the default 0.5 threshold, logistic regression produced much higher recall (**0.797**) but lower precision (**0.401**), while the random forest produced higher precision (**0.696**) but lower recall (**0.317**). In addition, the script generates calibration curves, which are essential when predicted probabilities are used as risk scores. A well-calibrated model supports more defensible threshold selection and escalation rules.

To assess generalization risk, I computed segment-wise performance metrics using `wine_type` as a segment identifier. This surfaces whether a model performs differently across meaningful groups (analogous to different product lines or processing conditions). The results demonstrate why this is necessary: for the random forest model, the ROC-AUC is **0.922** on the red segment (PR-AUC **0.730**, n=275) versus ROC-AUC **0.853** on the white segment (PR-AUC **0.602**, n=789). Such segmentation is a practical proxy for deployment scenarios where data distributions may differ by facility, supplier, or recipe.

Finally, I implemented a basic drift check using Kolmogorov–Smirnov (KS) tests comparing train vs test feature distributions. While not a full production drift monitoring system, this is a clear and reproducible starting point that highlights which features may shift and therefore require monitoring or periodic revalidation. In this controlled train/test split, drift is low as expected; the largest KS statistic observed was **0.043** (for `volatile_acidity`), with non-significant p-values. All artifacts are saved week-wise as CSV/PNG outputs and are suitable for GitHub publication as evidence of executed evaluation and risk-oriented thinking.
