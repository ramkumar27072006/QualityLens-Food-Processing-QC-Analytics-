In Week 2, I implemented the data cleaning and preprocessing stage as an executable pipeline, producing both a cleaned dataset and an auditable cleaning report. The primary goal was to convert the raw public dataset into a consistent, analysis-ready form while preserving traceability of every decision. This is essential in food processing contexts because cleaning choices can materially affect quality conclusions and any downstream predictive models.

The Week 2 workflow loads the raw dataset, applies type fixes (e.g., ensuring the quality label is an integer), removes exact duplicate rows, and performs domain-consistency checks that reflect the spirit of food processing validation. For example, the pipeline flags logically inconsistent sulfur dioxide relationships (free sulfur greater than total sulfur), rather than silently “fixing” records without documentation. In this dataset, the sulfur rule is satisfied (0 rows flagged), but the check is still important as a reusable validation pattern for real production data. In addition, I added robust outlier exploration using an IQR-based approach. Importantly, this process **flags** potential outliers and reports counts per feature, instead of automatically removing extreme values; in manufacturing data, extremes can represent true process deviations and should not be discarded without justification.

The executed cleaning results are captured in a structured JSON report. Concretely, the raw combined dataset contained **6,497 rows**, with **1,177 exact duplicate rows** detected. After deduplication, the cleaned dataset contains **5,320 rows** and remains complete with **0 missing values**, so no imputation was required. The cleaned segment sizes are also recorded (3,961 white and 1,359 red), which is useful for later segment reliability checks.

Outlier exploration is reported per variable using IQR thresholds, again as flagging-only evidence. The highest outlier counts were observed in `fixed_acidity` (304), `volatile_acidity` (279), and `chlorides` (237), with additional flags in `sulphates` and `citric_acid`. These figures provide practical guidance for feature engineering and robust modeling: they indicate where heavy tails and extreme values may affect scaling choices and model sensitivity.

The outputs of this week include (1) a cleaned dataset saved under `data/processed/`, (2) a structured JSON cleaning report containing counts of duplicates removed, missingness before/after, and outlier counts, and (3) a week-wise visualization that highlights the features with the most flagged outliers. These artifacts provide the required preprocessing blueprint, are reproducible, and are organized for GitHub publication as evidence of real work, not only a written plan.
